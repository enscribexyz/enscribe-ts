# AGENTS.md

## Purpose

This repository publishes `@enscribe/enscribe`, a TypeScript ESM library for naming deployed smart contracts with ENS.  
The core flow is orchestrated by `nameContract()` in `src/naming.ts`:

1. Detect contract type (`Ownable` or `ReverseClaimer`)
2. Create ENS subname if needed
3. Set forward resolution
4. Set reverse resolution

Use this file as the source of truth for AI coding agents making changes in this repo.

## Scope Boundaries

- This package handles ENS naming transactions and related detection logic.
- This package does **not** handle contract deployment orchestration.
- Public API is package-root exports from `src/index.ts` only.

## Canonical Source Of Truth

When code and docs disagree, trust these files first:

- Public exports: `src/index.ts`
- Runtime option/result contracts: `src/types.ts`
- Network/address config: `src/contracts.ts`
- Naming orchestration and behavior: `src/naming.ts`, `src/L1Naming.ts`, `src/L2Naming.ts`

## Public API Guardrails

Only the following runtime functions are currently exported from package root:

- `nameContract`
- `isContractOwner`
- `isOwnable`
- `isReverseClaimable`
- `getNetworkInfo`
- `getContractAddresses`

Do not document or depend on non-exported helpers as package-root API unless you also update `src/index.ts`.

## Documentation Consistency Rules

Before finishing any change:

- Ensure README API options match `NameContractOptions` in `src/types.ts`.
- Ensure README utility functions list matches actual package-root exports in `src/index.ts`.
- If network support changes, update both `src/contracts.ts` and README supported networks section.

## Network Configuration Caveats

- Supported network keys are defined by `NetworkName` in `src/contracts.ts`.
- Some L2 entries intentionally include blank optional addresses (`PUBLIC_RESOLVER`/`NAME_WRAPPER`) and should be treated as partial support.
- `ENS_REGISTRY` is required and validated; optional addresses can be empty.

## Build And Verification

Run these before requesting review or release:

- `pnpm run build`
- Optional smoke checks (require env setup):  
  - `pnpm run example`
  - `pnpm run example:ownable`

There is no dedicated test or lint script in this repo today. Keep changes minimal, typed, and behavior-safe.

## Change Checklist For Agents

- If adding/changing options or result fields:
  - Update `src/types.ts`
  - Update implementation call sites
  - Update README usage + API section
- If adding/changing exports:
  - Update `src/index.ts`
  - Update README API examples
- If changing chain logic:
  - Update `src/contracts.ts` and any chain-mapping logic
  - Verify supported chain names remain consistent

## Release Hygiene

- Package output is `dist/*` generated by `tsc`.
- `prepublishOnly` runs `clean` then `build`; do not rely on unstaged artifacts.
- Keep `README.md` accurate because it is shipped with the package.
